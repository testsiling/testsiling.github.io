    237.Delete Node in a Linked List

    I don’t like this one. The description is confusing and it doesn’t really need to delete a node, because it even didn’t provide the previous node, but asking “how to turn the current representative points to the next node”

    So the input node is [5, 1, 9], it needs [1, 9]

    Hence, the process would be :

    1. [5, 1, 9] -> [1, 1, 9]
    2. [1, 1, 9] -> [1, 9]

    Time and space complexity are both O(1). Anyway, this one is meaningless. I disliked it.

    160.Intersection of Two Linked Lists

    The description of this question is not well-stated. For instance, in example 1, why the intersection is 8, not1? The answer is that these two nodes, which value = 1, are pointed to different references. So the key here is to compare the reference, instead of simply compare the value.

    BTW, Brute Force will TLE.

    Hash Table

    Time: O(m+n)

    Space: O(m) + O(n)

    Two Pointers

    Connecting 2 linked lists and traverse them by 2 pointers. Elements after intersction would be the same if there is a intersection, or else there is no intersection. Example:

    A: [1,2,3,4,5]

    B: [9,8,7,3,4,5]

    A+B would be : [1,2,3,4,5,9,8,7,3,4,5]

    B+A would be : [9,8,7,3,4,5,1,2,3,4,5]

    once pA and pB is the same, reach the intersection, or reach None if there is no interection.
