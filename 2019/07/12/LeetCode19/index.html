224. Basic Calculator

  • Since some numbers have multiple digits, like 199, cannot just iterate the whole string. need to clean it up first.

  • Be careful with empty input, like " “.

  • Adding a pair if Brackets to the like would make it easier.

  • Be aware of the calculating logic using stack. (7+8-9) -> )9-8+7( -> (7+8-9)

  • some special cases: “2-(5-6)”, “1-(15)”

227. Basic Calculator II

Be careful with the case: 10-3/2. Directly calculate “-3/2” will return -2 in python, which is wrong. should treat is as “- (3/2)”

772. Basic Calculator III

Recursion

Just a combination of #227 and #224

sign is the last sign it met. num is the number after that sign

When meets left bracket, calculate the value inside and gent the length of the inner exprassion.

Iteration

When encounters the left bracket, save the current sign to the stack. then reset num and sign, continue to calculate the rest, until meets the right bracket. then sum the result in the stack before the last saved sign.

394. Decode String

The first idea comes into my mind is recursion. everytime meets [, jump into it, meets ], return. Using offset to skip the finished part.
